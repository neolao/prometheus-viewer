# Module: metrics

**Role:** Fetches and displays the list of metrics exposed by the selected machine on the connected Prometheus server, with loading, empty (machine-specific message) and error states. Once loaded, the list can be further narrowed client-side by a case-insensitive text search, with a distinct "no match" message when nothing fits. Clicking a metric name reveals, inline below that metric, both its current value and a graph of its evolution over time; only one metric's detail is shown at a time, and the selection resets whenever the machine (and thus the list) changes.
**Files:** `src/features/metrics/MetricList.tsx`, `src/features/metrics/MetricValue.tsx`, `src/features/metrics/MetricChart.tsx`
**Exports:**
- `MetricList({ baseUrl: string, machine: string })` — React component; re-fetches whenever `baseUrl` or `machine` changes.
- `MetricValue({ baseUrl: string, machine: string, metricName: string })` — React component, rendered by `MetricList` for the clicked metric; fetches and displays that metric's current value(s) on the machine, with loading, one-line-per-series success, "no current value" and error states; re-fetches whenever any prop changes.
- `MetricChart({ baseUrl: string, machine: string, metricName: string })` — React component, rendered by `MetricList` alongside `MetricValue` for the clicked metric; renders the metric's evolution over time as an inline SVG line chart (one polyline per series), with a "Time range" selector (last hour / 6h / 24h / 7 days, default last hour, each reloading the graph) plus a "Custom" option revealing start/end datetime inputs and an "Apply" button — no request is made for a custom range until applied; has loading, "no data for the selected range" and error states.
**Depends on:** [`modules/api-prometheus.md`](api-prometheus.md)
