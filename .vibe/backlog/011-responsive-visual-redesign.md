# Responsive Visual Redesign

## Description
The UI is functional but visually rough and desktop-oriented. Redesign the whole interface with polished, consistent styling — typography, spacing, colors, and loading/error/empty states — and make it fully responsive so it is pleasant to use on mobile, tablet, and desktop.

## Acceptance Criteria
- [ ] The app is usable on a mobile viewport (~375px wide) with no horizontal scrolling
- [ ] The layout adapts to tablet and desktop breakpoints (navigation, metric list, and detail views reflow appropriately)
- [ ] All existing views (machine selection, metric list, current value, evolution chart) share consistent typography, spacing, and colors
- [ ] Loading, error, and empty states remain readable and well laid out at every breakpoint

## Notes
Applies to existing views; new features (dashboard, favorites) should adopt the same design language when built. No specific CSS framework is imposed — keep the approach consistent with the current plain-CSS setup unless a decision record says otherwise.
